# Locale Template Reference

Use these templates when generating starter JSON files. Always generate BOTH locales simultaneously, with matching keys.

## next-intl ICU format equivalents

When generating `messages/{locale}.json` for next-intl, convert keys to ICU:

```json
{
  "common": {
    "itemCount": "{count, plural, one {# item} other {# items}}",
    "welcomeUser": "Welcome, {name}!",
    "validationMinLength": "Must be at least {min} characters"
  },
  "auth": { ... },
  "errors": { ... }
}
```

Note: next-intl uses a **single flat file per locale** with namespaces as top-level keys, vs i18next which uses **one file per namespace per locale**.

## CLDR plural rules quick reference

| Language | Categories used                  | Rule summary                              |
| -------- | -------------------------------- | ----------------------------------------- |
| English  | one, other                       | 1 → one; else other                       |
| Hindi    | one, other                       | 1 → one; else other                       |
| Gujarati | one, other                       | 1 → one; else other                       |
| Spanish  | one, other                       | 1 → one; else other                       |
| French   | one, many, other                 | 0–1 → one; 1e6 → many; else other         |
| German   | one, other                       | 1 → one; else other                       |
| Arabic   | zero, one, two, few, many, other | Full 6-form                               |
| Russian  | one, few, many, other            | Complex; use i18next-icu or next-intl ICU |
